1. 프로젝트 개요
------------------------------------------------------------
- 프로젝트명: Picklog (인스타그램 장소 아카이빙 서비스)
- 역할: 백엔드 API 서버
- 기술 스택: FastAPI + SQLAlchemy 2.x + PostgreSQL(PostGIS) + Playwright
- 배포: Railway (API) + Supabase (DB)


2. 인프라 구성 (완료)
------------------------------------------------------------
[로컬 개발 환경]
  - Docker Compose: PostgreSQL 15 + PostGIS 3.3, Redis
  - Poetry 기반 의존성 관리 (Python >=3.14)
  - Alembic 마이그레이션 관리

[클라우드 배포]
  - DB: Supabase (PostgreSQL + PostGIS)
    - Session Pooler 방식 (IPv4 지원)
    - 연결: aws-1-ap-southeast-1.pooler.supabase.com
  - API: Railway
    - GitHub 연동 자동 배포 (main 브랜치 push 시)
    - Dockerfile 기반 빌드
    - 도메인: capstone2be-production.up.railway.app
    - Swagger UI: https://capstone2be-production.up.railway.app/docs


3. DB 스키마 (완료 - 12개 테이블)
------------------------------------------------------------
  users              - 사용자 (email, password, kakao_id, nickname, profile_image)
  storages           - 저장소/창고 (title, description, is_public, deleted_at)
  storage_members    - 저장소 멤버 (storage_id, user_id, role: owner/editor/viewer)
  places             - 장소 마스터 (name, address, coordinate(PostGIS POINT), category_group, phone, homepage_url)
  place_raw_data     - 장소 원천 데이터 (provider, raw_payload(JSONB))
  place_images       - 장소 이미지 (image_url, source, is_representative)
  spots              - 스팟 (storage_id, place_id, added_by, instagram_url, user_memo, user_rating, is_visited)
  place_space_dna    - 장소 공간DNA (mbti_axes(JSONB), ai_summary)
  tags               - 태그 마스터 (name)
  place_tags         - 장소-태그 연결 (place_id, tag_id, score)
  user_space_dna     - 유저 공간DNA (mbti_axes(JSONB), preferred_vibe_tags, total_visits)
  user_space_dna_history - 유저 DNA 변화 이력 (spot_id, mbti_axes_snapshot)

핵심 설계 패턴:
  - StorageMember 다대다 관계로 공유 창고 지원 (owner/editor/viewer 역할)
  - Place-first 정규화: 장소 정보를 Place에 중앙 관리, Spot은 개인 저장 기록
  - 소프트 삭제: storages.deleted_at, spots.deleted_at
  - GeoAlchemy2 POINT + GIST 인덱스 (idx_places_coordinate 자동 생성)


4. 구현된 API 엔드포인트 (완료)
------------------------------------------------------------
[인증 - /auth]
  POST /auth/register       회원가입 (기본 저장소 자동 생성)
  POST /auth/login          로그인 (OAuth2 form data, JWT 발급)

[사용자 - /users]
  GET  /users/me            내 정보 조회
  PUT  /users/me            프로필 수정 (nickname, profile_image)

[저장소 - /storages]
  GET    /storages                   내 저장소 목록 (페이징: page, size)
  POST   /storages                   저장소 생성 (owner 자동 등록)
  GET    /storages/{id}              저장소 상세 조회
  PUT    /storages/{id}              저장소 수정 (owner, editor)
  DELETE /storages/{id}              저장소 삭제 (소프트 삭제, owner만)

[스팟 - /storages/{storage_id}/spots]
  GET    /storages/{id}/spots        스팟 목록 (페이징: page, size)
  POST   /storages/{id}/spots        스팟 생성 (place_id 필수)
  GET    /storages/{id}/spots/{sid}  스팟 상세
  PUT    /storages/{id}/spots/{sid}  스팟 수정 (is_visited=True 시 visited_at 자동 기록)
  DELETE /storages/{id}/spots/{sid}  스팟 삭제 (소프트 삭제)

[장소 - /places]
  GET  /places?q=검색어              장소 검색 (이름 기반, 페이징)
  GET  /places/{id}                  장소 상세 조회

[인스타그램 - /instagram]
  POST /instagram/crawl      URL 크롤링 (OG 메타 기반 캡션/이미지/위치 추출)
  POST /instagram/save       크롤링 + 저장 (Place -> PlaceRawData -> PlaceImage -> Spot)

[헬스체크 - /health]
  GET  /health/db            DB 연결 상태 확인


5. 주요 구현 특징
------------------------------------------------------------
- JWT 인증: OAuth2PasswordBearer + Swagger Authorize 버튼 정상 동작
- 권한 관리: StorageMember 기반 role 검사 (owner/editor/viewer)
- Instagram 크롤링: Playwright + BeautifulSoup OG 메타 파싱
  - URL 검증 (instagram.com 여부)
  - 비공개/삭제 게시물 감지 (OG 메타 비어있으면 404)
  - 타임아웃/차단 처리 (504)
- 페이징: GET /storages, GET /spots에 page/size 쿼리 파라미터
- 소프트 삭제: deleted_at 필드 활용, 목록 조회 시 자동 필터
